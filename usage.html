thrive-slides 使い方マニュアル

thrive-slides は AIエージェント非依存のスライド・デザインシステムです。特定のベンダーや環境に縛られず、ファイルを読めてシェルを実行できるエージェントであれば、どれでも利用できます。呼び出し方は使う環境によって異なります。まず下の早見表で自分の環境を確認してください。

早見表（環境別の呼び出し方）

環境 使い方 入口ファイル
Claude Code（ターミナル / VS Code・JetBrains 拡張） /thrive-slides で明示起動、または依頼内容から自動起動 SKILL.md
claude.ai チャット（ブラウザ） 「スライドを作って」等と伝えると、スキルの説明文に基づき自動起動 SKILL.md（スキルとして配布）
Claude Design（claude.ai の Design タブ） スキルではなくファイル添付で使う design/CLAUDE-DESIGN-GUIDE.md
Codex CLI（OpenAI Codex CLI） AGENTS.md を自動読み込み → references/ を文脈に → 仕様準拠生成またはレンダラ実行 AGENTS.md
Cursor .cursor/rules/thrive-slides.mdc が自動ロード → 同様に利用 .cursor/rules/thrive-slides.mdc
ChatGPT（Custom GPT / Projects） chatgpt/ バンドルをアップロードして使う（シェル不要） chatgpt/SYSTEM-PROMPT.md + chatgpt/knowledge/
MCP 対応ホスト全般（Claude Desktop / Claude Code / Cursor / Cline 等） mcp/ の MCP サーバを登録し、仕様取得＋HTML/PPTX レンダリングをツールとして呼び出す mcp/README.md
汎用（どんなエージェントでも） references/ を読んで仕様準拠生成 references/generation-guide.md

パターン名（例: three-kpi-big-number）を指示に使うと、狙ったレイアウトを指定できます。全83パターンの名前とサムネイルは公開カタログで一覧確認できます → https://ry-from-garage.github.io/thrive-slides-page/ （各スライドの左上に pattern: 名前 が表示されます）。

1. Claude Code（ターミナル / IDE 拡張）

スラッシュコマンドが使えるのはこの環境です。

インストール

スキルフォルダを ~/.claude/skills/ に置きます。

git clone https://github.com/ry-from-garage/thrive-slides.git
cp -R thrive-slides ~/.claude/skills/thrive-slides

~/.claude/skills/thrive-slides/SKILL.md がある状態になれば OK です（VS Code / JetBrains の Claude Code 拡張でも同じ場所を参照します）。

呼び出し方（2通り）

  • スラッシュコマンド（明示）: メッセージの先頭で /thrive-slides と入力し、続けて内容・テーマを書きます。
  • 自動起動: 「スライドを作って」「このメモでデッキを作って」のように、スキルの説明文に合う依頼をすると、Claude が自動で thrive-slides を使います（スラッシュ不要）。

指定の例

/thrive-slides strategy-consulting テーマで、以下の内容から提案スライドを作ってください。

# 国内SaaS参入戦略
- 市場規模は4.2兆円（2030年予測）
- 競合は freee / マネーフォワード / 弥生
- 18か月で黒字化を目指す

指示の中で次を自由に指定できます。

指定したいこと 書き方の例
テーマ（5つから1つ） 「strategy-consulting テーマで」「modern-tech で」 ※省略すると内容・目的から自動選択
レイアウト/パターン 「KPIは three-kpi-big-number で」「比較は harvey-balls-comparison で」 ※名前はカタログで確認
内容 箇条書き・文章・Markdown をそのまま貼り付け。.md / .pdf ファイルを渡してもOK
ブランド適用 記入済みの brand-<会社名>.md を渡して「このブランドを適用して」

2. claude.ai チャット（ブラウザ）

claude.ai のチャットにはスラッシュコマンドはありません。スキルは、依頼内容がスキルの説明文に一致したときに自動起動します。

  • スキルの用意: カスタムスキルの追加方法は契約形態により異なります（組織で配布されている場合は自動で利用可能になります）。個人で確実に試したい場合は、Claude Code（上の方法）が最も簡単です。
  • 使い方: テーマ名・パターン名・内容を文章で伝えるだけです。
thrive-slides の strategy-consulting テーマで、次のメモからスライドを作ってください。
（内容を貼り付け、または .md / .pdf を添付）

3. Claude Design（claude.ai の Design タブ）

Claude Design はスキルを使いません（チャット側で有効化したスキルは Design には引き継がれません）。ファイルを添付して使います。

添付するファイル

ファイル 場所 必須/任意
CORE（生成手順＋グリッド＋品質ルール） design/00-CORE-thrive-slides.md 必須
テーマ（5つから1つ） references/themes/theme-<選択>.md 必須
レイアウト（使うカテゴリ分） references/layouts/layout-<カテゴリ>.md 必須
ブランド references/brand/BRAND-TEMPLATE.md をコピーして記入 任意

指示の例

添付のファイルに従って strategy-consulting テーマでスライドを生成してください。
内容: …（メモ・箇条書き・Markdown など）

詳しい手順は Claude Design ガイド を参照してください。

4. Codex CLI（OpenAI Codex CLI）

Codex CLI はプロジェクトルートの AGENTS.md を自動的に読み込みます。追加設定なしで thrive-slides の使い方が文脈に入ります。

使い方

AGENTS.md の「起動手順（Codex / Cursor — MCP なし）」に従ってスライドを生成します。

  1. AGENTS.md（このリポジトリの /AGENTS.md）を確認する（Codex が自動で読み込む）
  2. references/generation-guide.md を読む（生成規範・品質ルール）
  3. references/themes/THEME-INDEX.md でテーマを選び、対応する theme-*.md を読む
  4. references/layouts/LAYOUT-INDEX.md でレイアウトを選び、対応する layout-*.md を読む
  5. 仕様に従ってスライドを生成するか、レンダラを実行する

レンダラ実行（任意）

# HTML デッキ
cd renderer && npm install && node build.js <spec.json> <out>

# PowerPoint
cd pptx && pip install -r requirements.txt && python build.py <spec.json> <out.pptx>

spec フォーマットは renderer/specs/gallery.json を参照してください。

5. Cursor

.cursor/rules/thrive-slides.mdc がプロジェクト内に存在する場合、Cursor が自動的にこのルールをロードします。スライド/デッキ生成を依頼すると、このルールに基づいて thrive-slides の仕様に沿った生成が行われます。

使い方

Cursor のチャットまたはエディタ内で、スライド生成を依頼するだけです。

  • ルールが自動ロードされるため、特別な設定なしに references/ を参照した仕様準拠生成が実行されます。
  • レンダラの実行（renderer/build.js や pptx/build.py）も Cursor のターミナルから行えます。
  • MCP サーバを登録することで、ツールとしての呼び出しも可能です（→ 後述の「MCP 対応ホスト」を参照）。

6. ChatGPT（Custom GPT / Projects）

ChatGPT はローカルのレンダラを実行できません。chatgpt/ バンドルをアップロードすることで、ChatGPT 上で thrive-slides のデザイン規範に沿ったスライドを生成できます。

セットアップ

詳細な手順は chatgpt/README.md を参照してください。概要:

Custom GPT を作る場合:

  1. ChatGPT を開く → 左サイドバーの「GPTs」→「+ 作成」をクリック
  2. 「構成 (Configure)」タブを選択
  3. Instructions（指示） に chatgpt/SYSTEM-PROMPT.md の内容を貼り付ける
  4. 「知識 (Knowledge)」 に以下の4ファイルをアップロードする:
    • chatgpt/knowledge/00-core.md
    • chatgpt/knowledge/01-themes.md
    • chatgpt/knowledge/02-layouts.md
    • chatgpt/knowledge/03-brand.md
  5. 保存して完成

ChatGPT Projects を使う場合:

  1. 左サイドバーの「Projects」→「+ 新しいプロジェクト」をクリック
  2. プロジェクト指示に chatgpt/SYSTEM-PROMPT.md の内容を貼り付ける
  3. chatgpt/knowledge/ 内の4ファイルをアップロードする

出力

ChatGPT は以下の2つを出力します。

  • 自己完結 HTML デッキ — ブラウザで直接開いてプレゼンできる HTML ファイル（テーマカラーをインライン CSS で再現）
  • デッキ仕様 JSON — {title, theme, slides: [{page, pattern, action_title, content}]} 形式。この JSON を MCP サーバや pptx/build.py・renderer/build.js に渡すと、ピクセル精度のデッキを生成できます。

7. MCP 対応ホスト全般

thrive-slides は MCP（Model Context Protocol）サーバとしても使えます。mcp/ に stdio の MCP サーバを同梱しており、Claude Desktop / Claude Code / Cursor / Cline / ChatGPT デスクトップコネクタなど 任意の MCP 対応ホストから first-class なツールとして利用できます。

MCP サーバが提供するツール（全9つ）

仕様ツール（7つ） — 設計仕様をホストの文脈に取り込む

ツール 説明
list_themes 5テーマの名前と概要、THEME-INDEX.md を返す
list_patterns 14カテゴリ・全パターン名を返す
get_generation_guide 生成規範・品質ルール（references/generation-guide.md）を返す
get_grid_system グリッドシステム定義（12カラム・8px・1280×720px）を返す
get_theme 指定テーマの定義ファイル（theme-<name>.md）を返す
get_layout 指定カテゴリのレイアウト定義（layout-<category>.md）を返す
get_brand_template ブランドテンプレート＋融合規則を返す

レンダリングツール（2つ） — デッキ spec から完成品を生成

ツール 説明
render_html spec オブジェクトを受け取り、HTML デッキを生成して絶対パスを返す
render_pptx spec オブジェクトを受け取り、PowerPoint ファイルを生成して絶対パスを返す（デフォルトでフォント埋め込み済み）

セットアップ

cd mcp && npm install

spec ツール（7つ）はこれだけで使えます。レンダリングツールを使う場合は renderer / pptx の依存関係も別途インストールしてください（詳細は mcp/README.md）。

ホストへの登録方法

Claude Code:

claude mcp add thrive-slides -- node /絶対パス/thrive-slides/mcp/server.js

（-s user でユーザーグローバル登録、-s project でプロジェクト共有）

Claude Desktop — claude_desktop_config.json に追加:

{
  "mcpServers": {
    "thrive-slides": {
      "command": "node",
      "args": ["/絶対パス/thrive-slides/mcp/server.js"]
    }
  }
}

Cursor / Cline / その他 — ホスト別の設定ファイルへの追記方法は mcp/README.md を参照してください。

使用の流れ（例）

  1. list_patterns を呼び出してパターン一覧を確認する
  2. get_theme で使用するテーマの定義を取得する
  3. get_layout で必要なレイアウトカテゴリの定義を取得する
  4. spec JSON を組み立てる（renderer/specs/gallery.json 参照）
  5. render_pptx に spec を渡して PowerPoint を生成する

8. 汎用（どんなエージェントでも使える）

シェルさえ実行できれば、任意のエージェントで thrive-slides を利用できます。

  1. references/generation-guide.md を読む（生成規範・品質ルール）
  2. references/themes/THEME-INDEX.md でテーマを選び、対応する theme-*.md を読む
  3. references/layouts/LAYOUT-INDEX.md でレイアウトを選び、対応する layout-*.md を読む
  4. 仕様に従ってスライドを生成する（レンダラ実行は任意）

詳しくは AGENTS.md（リポジトリルート）を参照してください。

PPTX（PowerPoint）として出力する

これは何か

pptx/ チャネルは renderer/specs/ の仕様に基づき、同じ83パターンをネイティブ PowerPoint (.pptx) として出力する Python パイプラインです。HTML レンダラと同じ gallery.json を入力に使います。

ダウンロード（すぐ試す）

全98スライドの見本デッキを直接ダウンロードできます。

  • URL: https://ry-from-garage.github.io/thrive-slides-page/thrive-slides-gallery.pptx
  • または: 公開ギャラリー（https://ry-from-garage.github.io/thrive-slides-page/）上部の「⬇ PowerPoint版」ボタンから

日本語フォント（Noto Sans JP / Noto Serif JP）埋め込み済みなので、Noto フォントがインストールされていない環境でも同じ見た目で開けます。

自分で生成する手順

cd pptx
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/python build.py            # renderer/specs/gallery.json → out/gallery.pptx（全98スライド）
# 任意のスペックから生成する場合:
.venv/bin/python build.py <spec.json> <out.pptx>

日本語フォント埋め込み（環境差の解消）

.venv/bin/python embed_fonts.py out/gallery.pptx out/gallery-embedded.pptx

embed_fonts.py は Noto Sans JP / Noto Serif JP をデッキ使用文字だけにサブセットして OOXML で正規埋め込みします（PowerPoint の「使用文字のみ埋め込み」と同等）。これで Noto 未インストールの PowerPoint でも同じ見た目になります。

自分のスライドを作る

入力スペックは renderer/specs/gallery.json と同じ形式（slides: [{theme, pattern, content, page}]）です。theme は5種から、pattern は全83パターン名（公開カタログで確認）から選びます。

困ったときは

  • どのレイアウトがあるか分からない → 公開カタログ（https://ry-from-garage.github.io/thrive-slides-page/ ）でサムネイルとパターン名を確認。
  • 用語が分からない → README 最下部の用語集。
  • スラッシュコマンドが出てこない → claude.ai チャット / Claude Design / Codex / Cursor / ChatGPT にはありません（Claude Code のみ）。上の早見表で環境を確認してください。
  • MCP サーバが接続できない → mcp/README.md のトラブルシューティングを参照してください。サーバは stderr のみにログを出力します（stdout は JSON-RPC チャンネルです）。
