thrive-slides 使い方マニュアル

thrive-slides は、使う環境によって呼び出し方が異なります。以下の3通りです。まず下の早見表で自分の環境を確認してください。

早見表（環境別の呼び出し方）

環境 呼び出し方 スラッシュコマンド
Claude Code（ターミナル / VS Code・JetBrains 拡張） /thrive-slides で明示的に呼び出す、または依頼内容から自動起動 ✅ あり（/thrive-slides）
claude.ai チャット（ブラウザ） 「スライドを作って」等と伝えると、スキルの説明文に基づき自動起動 ❌ なし（自動起動のみ）
Claude Design（claude.ai の Design タブ） スキルではなくファイル添付で使う ❌ なし

パターン名（例: three-kpi-big-number）を指示に使うと、狙ったレイアウトを指定できます。全83パターンの名前とサムネイルは 公開カタログ で一覧確認できます → https://ry-from-garage.github.io/thrive-slides-page/ （各スライドの左上に pattern: 名前 が表示されます）。

1. Claude Code（ターミナル / IDE 拡張）

スラッシュコマンドが使えるのはこの環境です。

インストール

スキルフォルダを ~/.claude/skills/ に置きます。

git clone https://github.com/ry-from-garage/thrive-slides.git
cp -R thrive-slides ~/.claude/skills/thrive-slides

~/.claude/skills/thrive-slides/SKILL.md がある状態になればOKです（VS Code / JetBrains の Claude Code 拡張でも同じ場所を参照します）。

呼び出し方（2通り）

  • スラッシュコマンド（明示）: メッセージの先頭で /thrive-slides と入力し、続けて内容・テーマを書きます。
  • 自動起動: 「スライドを作って」「このメモでデッキを作って」のように、スキルの説明文に合う依頼をすると、Claude が自動で thrive-slides を使います（スラッシュ不要）。

指定の例

/thrive-slides strategy-consulting テーマで、以下の内容から提案スライドを作ってください。

# 国内SaaS参入戦略
- 市場規模は4.2兆円（2030年予測）
- 競合は freee / マネーフォワード / 弥生
- 18か月で黒字化を目指す

指示の中で次を自由に指定できます。

指定したいこと 書き方の例
テーマ（5つから1つ） 「strategy-consulting テーマで」「modern-tech で」 ※省略すると内容・目的から自動選択
レイアウト/パターン 「KPIは three-kpi-big-number で」「比較は harvey-balls-comparison で」 ※名前はカタログで確認
内容 箇条書き・文章・Markdown をそのまま貼り付け。.md / .pdf ファイルを渡してもOK
ブランド適用 記入済みの brand-<会社名>.md を渡して「このブランドを適用して」

2. claude.ai（ブラウザのチャット）

claude.ai のチャットには スラッシュコマンドはありません。スキルは、依頼内容がスキルの説明文に一致したときに自動起動します。

  • スキルの用意: カスタムスキルの追加方法は契約形態により異なります（組織で配布されている場合は自動で利用可能になります）。個人で確実に試したい場合は、Claude Code（方法1）が最も簡単です。
  • 使い方: テーマ名・パターン名・内容を文章で伝えるだけです。
thrive-slides の strategy-consulting テーマで、次のメモからスライドを作ってください。
（内容を貼り付け、または .md / .pdf を添付）

3. Claude Design（claude.ai の Design タブ）

Claude Design は スキルを使いません（チャット側で有効化したスキルは Design には引き継がれません）。ファイルを添付して使います。

添付するファイル

ファイル 場所 必須/任意
CORE（生成手順＋グリッド＋品質ルール） design/00-CORE-thrive-slides.md 必須
テーマ（5つから1つ） references/themes/theme-<選択>.md 必須
レイアウト（使うカテゴリ分） references/layouts/layout-<カテゴリ>.md 必須
ブランド references/brand/BRAND-TEMPLATE.md をコピーして記入 任意

指示の例

添付のファイルに従って strategy-consulting テーマでスライドを生成してください。
内容: …（メモ・箇条書き・Markdown など）

詳しい手順は Claude Design ガイド を参照してください。

PPTX（PowerPoint）として出力する

これは何か

pptx/ チャネルは renderer/specs/ の仕様に基づき、同じ83パターンを ネイティブ PowerPoint (.pptx) として出力する Python パイプラインです。HTML レンダラと同じ gallery.json を入力に使います。

ダウンロード（すぐ試す）

全98スライドの見本デッキを直接ダウンロードできます。

  • URL: https://ry-from-garage.github.io/thrive-slides-page/thrive-slides-gallery.pptx
  • または: 公開ギャラリー（https://ry-from-garage.github.io/thrive-slides-page/）上部の「⬇ PowerPoint版」ボタンから

日本語フォント（Noto Sans JP / Noto Serif JP）埋め込み済みなので、Noto フォントがインストールされていない環境でも同じ見た目で開けます。

自分で生成する手順

cd pptx
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/python build.py            # renderer/specs/gallery.json → out/gallery.pptx（全98スライド）
# 任意のスペックから生成する場合:
.venv/bin/python build.py <spec.json> <out.pptx>

日本語フォント埋め込み（環境差の解消）

.venv/bin/python embed_fonts.py out/gallery.pptx out/gallery-embedded.pptx

embed_fonts.py は Noto Sans JP / Noto Serif JP をデッキ使用文字だけにサブセットして OOXML で正規埋め込みします（PowerPoint の「使用文字のみ埋め込み」と同等）。これで Noto 未インストールの PowerPoint でも同じ見た目になります。

自分のスライドを作る

入力スペックは renderer/specs/gallery.json と同じ形式（slides: [{theme, pattern, content, page}]）です。theme は5種から、pattern は全83パターン名（公開カタログで確認）から選びます。

AIエージェント別の使い方

thrive-slides は AIエージェント非依存です。Claude 以外のエージェントでも同じ仕様・レンダラを利用できます。

早見表（AIエージェント別）

エージェント 使い方
Claude Code（ターミナル / IDE 拡張） SKILL.md が自動読み込み。/thrive-slides で明示起動 or 自動起動
claude.ai チャット 依頼内容からスキルが自動起動（スラッシュコマンドなし）
Claude Design ファイル添付方式（design/CLAUDE-DESIGN-GUIDE.md 参照）
Codex（OpenAI Codex CLI） AGENTS.md を自動読み込み。references/ を文脈に入れてレンダラ実行 or 仕様準拠生成
Cursor / Aider / その他のコーディングエージェント AGENTS.md をプロジェクトルールとして読み込み、同様に利用
汎用（どんなエージェントでも） references/generation-guide.md → テーマ → レイアウトの順に読み、仕様に従って生成

汎用（どんなエージェントでも使える）パス

  1. references/generation-guide.md を読む（生成規範・品質ルール）
  2. references/themes/THEME-INDEX.md でテーマを選び、対応する theme-*.md を読む
  3. references/layouts/LAYOUT-INDEX.md で使うレイアウトを選び、対応する layout-*.md を読む
  4. 仕様に従ってスライドを生成する（レンダラ実行は任意）

詳しくは AGENTS.md（リポジトリルート）を参照してください。

出力形式

デフォルトの出力は HTML（ブラウザ表示） ですが、PowerPoint（.pptx）でも出力できます。pptx/ フォルダで python build.py を実行すると、全83パターンをネイティブ .pptx として書き出せます。詳しい手順は README の「PowerPoint 出力（pptx/）」 を参照してください。

困ったときは

  • どのレイアウトがあるか分からない → 公開カタログ（https://ry-from-garage.github.io/thrive-slides-page/ ）でサムネイルとパターン名を確認。
  • 用語が分からない → README 最下部の用語集。
  • スラッシュコマンドが出てこない → claude.ai チャット / Claude Design にはありません（Claude Code のみ）。上の早見表で環境を確認してください。
